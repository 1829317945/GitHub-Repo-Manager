# GitHub Repo Manager (CLI)

这是一个基于 `gh-cli` 的交互式 Bash 脚本，旨在简化 GitHub 仓库的日常管理。它不仅支持快速删除，还能在重命名远程仓库后，**自动搜索并更新本地所有关联的 Git 远程地址**。

## 🚀 核心功能

*   **智能选择**：自动检测 `fzf`。若已安装，开启模糊搜索及方向键选择；若未安装，回退至编号列表。
*   **一键重命名**：一键修改 GitHub 仓库名。
*   **本地同步（特色）**：重命名后，脚本会自动扫描 `$HOME` 目录下所有 `.git` 配置，将旧的远程 URL 批量更新为新 URL，解决手动重连的烦恼。
*   **交互式删除**：带确认机制的仓库清理工具。

## 🛠️ 前置条件

1.  **GitHub CLI**: 需安装 `gh` 并完成登录 (`gh auth login`)。
2.  **fzf** (推荐): 安装后可获得极佳的交互体验。
3.  **column/awk**: 通用 Linux 工具（通常预装）。

## 📦 安装与使用

# 克隆仓库
git clone git@github.com:1829317945/GitHub-Repo-Manager.git

# 进入目录
cd GitHub-Repo-Manager

# 赋予执行权限并运行
chmod +x manager.sh
./manager.sh

## 🖥️ 菜单预览

```text

4) 退出 (Exit)

## ⚠️ 注意事项

*   **重命名功能**：脚本会扫描 `$HOME` 目录下的 `.git/config`。如果你的项目存放在非常规路径，请自行调整脚本中的 `find` 路径。
*   **危险操作**：删除操作不可逆，请务必确认仓库名称后再确认。

---
**小提示**：建议将此脚本加入 `alias` 或移动到 `/usr/local/bin`，随时随地管理你的 GitHub 资产。
